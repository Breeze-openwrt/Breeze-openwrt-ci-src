## 🚀 本次“去 Node 化”大改造亮点 (性能狂魔特供版)

### 🔴 痛点是什么？
老旧的 Node.js 20 核心即将在 GitHub 停用。原本我们依赖的“历史包清理插件”的作者已经由于各种原因跑路。不论框架层怎么修改 `env` 指令强制降配，只要它调用了遗留依赖包的心跳程序，就会满屏飘红警告，折磨强迫症。

### 🟢 我们做了什么？ (极简架构重构)

为了彻底断绝被第三方依赖卡脖子的隐患，我们直接把那些冗余的旧插件**全部连根拔起**，引入原汁原味的官方 CLI 配合流线型 Shell 脚本，对多余记录实现降维打击。

#### 🛡️ 绝杀招式：超高速历史包自动清理的“流水线”美学
来看一段我们的神级清理代码（无任何附加依赖）：
```bash
  # 列出所有发版记录，跳过最顶端精华的 20 个，剩下的黑名单交由循环轰除
  tags=$(gh release list -L 100 | awk -F '\t' '{print $1}' | tail -n +21)
  for tag in $tags; do gh release delete "$tag" --cleanup-tag -y; done
```
**🧠 编程底层思想大揭秘（Linux 流畅艺术）**：
这短短 3 行实现了前人几千点下载量的冗长第三方组件功能！主要分为三个工段串联（管道符 `|`）：
- **提取员 (`gh release list`)**: 这是 Github 云端环境里默认预装的最佳管家。下达命令后秒速交出版本台账。它使用 Go 语言写成，纯血原生，绝无 Node 引擎报废危机。
- **切割师 (`awk`)**: Linux 的上古文本处理法宝。用一个制表符切开表格，只需取出第一栏的核心版本代号（黑名单猎物）。
- **护城河 (`tail -n +21`)**: 意法叫“从头斩断”。跳过榜单最前的 20 个最新发布（免责金牌），剩下的直接排遣输出结账。最后交由 `for` 循环精准爆破。

这样一番改造后，所有清理阶段的速度将如风驰电掣，永远不看环境变量的脸色！
